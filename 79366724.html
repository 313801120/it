  • 关于网上程序员接私活的问题

  • 作者： 发表于：2018-09-16 20:31:20
    文丨eachfeel
    很多人都谈网上接私活这件事，其实大家想法都差不多，现在做软件的那么多，工资也不是那么高了，不像以前学一门语言的人不是那么多，随便进个公司就很吃香，然后每天几个小时，有时候也常常会有点加班，但是会这个的人少，工资高，待遇好。现在会这语言的人太多了，早上上班，随便出个事故，像电梯逆行了之类的就会有一大批程序员向公司打电话，“XX经理，今天上班的时候地铁逆行了，我受伤了，XX项目太急了，你交给别人做吧，我今天做不了了，XX客户今天可能会打电话过来，你把我的电脑开一下吧，今天我在家远程做”，以前我也敢做这样的事。像以前上学的时候谈c,c++的时候都是不愿意学的，感觉没有java，.net学着简单，没有这两个工资高，现在完全反过来了，学c的 c++的和现在我们学java， .net的工资差的不是一点半点，牛逼一些的还是学c c++的，很多新技术的开发，新产品的出现，还是要c c++来完成。没听过一个操作系统是用.net来写的。不过这些还是可以用来接私活的，多的不好赚，赚点小外快还是好的，学asp jsp php java .net等等太多的人抢这碗饭吃。但是，我们还是想在这拥挤的人群中找点饭吃。
    关于接私活这事嘛，朋友介绍这个可能大家会放心一些，因为毕竟是熟人，但是也有可能，朋友介绍的这个人朋友个人也并不认识。也不是诋毁朋友的意思，只是自己的一些想法，不代表所有人。
    接私活的过程其实也是在累积人脉的过程，累积私活的过程，就像不知道哪天客户还会有别的需求要做，还有别的朋友也要做同样的东西等等。刚开始价格公道一点，自己做的想的周到细致一点，质量做精细一点也是好的。因为客户也可能会成为朋友。
    接这些私活的时候可能我们两方面都信不过，客户怕你拿着钱跑了结果什么都没得到，你怕客户找多个人来做，你只是其中一个备胎，他选定之后肯定没你什么事了。所以接私活的时候押金肯定是必须的，安全这方面也是必须的。
    1.客户要缴纳20%-50%押金，不给的话大可不接。
    2.项目交接完成后，和客户讲明自己不负责维护，也可能有自己疏忽的问题，这些是自己的责任，本着良心来说这是要免费来搞的，当然如果你约定好之后说项目结束后不是自己的问题了，别人也没法说什么，但是你在他心中分量发生了变化，以后还有没有类似的私活就很难说了。
    3.给客户看的时候尽量放在自己的平台上让他看一下，然后再谈满意度问题。
    4.客户要能写清文档，能表达出自己的什么意思，不要到时候做的和客户说的完全不是一个东西，这就造成了很尴尬的局面。讲清楚，文档敲定后，中途另外加功能有可能要收部分费用。
    5.后期维护期间新加功能要收费，这个也要和客户说清楚。
    6.交易的时候可以在一些第三方平台上完成，到时两方面都能放心，我目前多是在淘宝完成。
